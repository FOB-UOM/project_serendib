# Future Pillars Template

Use this template to add new pillars without changing the overall architecture.

## Steps

1. Copy `pillar-name-template/`.
2. Rename it to `pillar-<new-domain>/`.
3. Rename subfolders to visible sub-pillars.
4. Add a short `README.md` in each sub-pillar.
5. Update root `README.md`, `CONTRIBUTING.md`, and `platform/app.py` links.

The template is intentionally minimal so it can be reused for any future domain.
