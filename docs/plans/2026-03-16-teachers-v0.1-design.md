# Teachers v0.1 — Dataset design

## Scope

Build the first focused slice of `Lanka-Instruct-v1` for **Sri Lanka school teachers**, targeting daily workflows.

- **Audience**: teachers across Primary, O/L, A/L (mixed)
- **Language**: English + Singlish (Sinhala-in-Latin code-mix)
- **Assistant persona**: “Sri Lanka Teacher Assistant” (safe, practical, avoids PII)

## Priority task types (v0.1)

1. **Classroom management + pedagogy** (routines, engagement, behavior, inclusive strategies)
2. **Low-resource teaching aids** (no projector, minimal printing, chalkboard-first activities)
3. **Exam prep** (school tests, O/L/A/L style prep, revision plans, practice questions)
4. **Remediation / differentiation** (support weak students, mixed-ability classes, targeted practice)

## Target size and release shape

- **Target**: 1,000 verified dialogues
  - 250 per sub-domain (4 × 250)
- **Dialogue shape**: multi-turn (2–6 turns typical)
- **Outputs**: teacher-ready artifacts (plans, scripts, checklists, question sets, rubrics)

## Data model (existing repo schema)

Use the canonical JSONL format in `data/hf_dataset_template/`:

- `language`: `en` for clean English; `mixed` for Singlish
- `domain`: one of:
  - `teachers_classroom_mgmt`
  - `teachers_low_resource`
  - `teachers_exam_prep`
  - `teachers_remediation`

Each record uses `conversations[]` messages with roles `system|user|assistant`.

## Safety constraints

- Do **not** include student PII (names, addresses, phone numbers, IDs).
- Child-safety aware: avoid encouraging harmful discipline; prefer supportive interventions.
- Avoid medical/legal diagnosis; recommend seeking official guidance where relevant.

## Verification rubric (Argilla)

Labels to capture for each record:

- **Language**: `en` vs `mixed`
- **Domain**: one of the four domains above
- **Usefulness** (1–5): directly usable by teachers
- **Clarity** (1–5): actionable steps, minimal ambiguity
- **Safety/PII**: `none` | `suspected` | `removed`
- **Factuality risk**: `low` | `medium` | `high` (especially for exam policies / syllabus claims)
- **Decision**: `approve` | `needs_edit` | `reject`
- **Notes**: short justification + suggested fix

## Acceptance criteria (v0.1)

- 100% schema-valid JSONL
- 0 known PII leaks
- ≥80% records “approve” after review, or fixed via curation pass
- Dataset card includes limitations and intended use
